## What This Is

`sourcetrace` is an automated truth-extraction engine that points at any codebase and produces a structured reality map: what the system actually guarantees, where documentation lies, and where the footguns hide. It is the core analytical backbone of the cartograph/threshold ecosystem, ingesting raw source and emitting verified, structured intelligence about a codebase's actual behavior.

## Claimed Intellectual Property

1. **Reality-map extraction methodology** — the pipeline design for converting raw source code into a machine-verifiable, human-readable map of system guarantees, behavioral contracts, and known-unsafe patterns, without requiring human annotation.
2. **Documentation-drift detection** — the automated technique for identifying contradictions between documented behavior and observed code behavior, including staleness scoring and divergence flagging.
3. **Footgun surface detection** — the algorithmic identification and classification of dangerous code patterns, anti-patterns, and undocumented failure modes exposed by the codebase to its consumers.
4. **Source truth pack format** — the structured output schema produced by sourcetrace, representing a codebase's full behavioral contract as a portable, machine-consumable artifact consumed by downstream tools (cartograph, threshold).
5. **Truth-verification pipeline** — the end-to-end orchestration of static analysis, doc comparison, runtime-signature inference, and output normalization into a single deterministic run.
